### 问题1：前端导入语句包含不必要的文件扩展名

**文件位置**: `frontend/src/main.tsx:11`

**问题描述**: 
导入 App 组件时使用了 `.tsx` 扩展名，这不是 TypeScript 推荐的做法。

**原始代码**:
```typescript
import App from './App.tsx'
```

**修复后代码**:
```typescript
import App from './App'
```

**修复说明**:
- TypeScript 导入语句不需要显式包含文件扩展名
- 移除 `.tsx` 扩展名后，TypeScript 编译器会自动解析正确的文件
- 这符合 TypeScript 的标准导入规范

### 问题2：后端代码中使用 print 语句进行调试输出

**文件位置**: `backend/api/v1/projects.py:906-970`

**问题描述**: 
在 `get_processing_status` 函数中使用了 `print` 语句进行调试输出，并且直接输出到 `sys.stderr`。这不是生产环境推荐的做法。

**原始代码**:
```python
import sys
print(f"DEBUG ENTRY get_processing_status: project_id = {project_id}", file=sys.stderr, flush=True)
try:
    print(f"DEBUG: processing_service = {processing_service}, type = {type(processing_service)}", file=sys.stderr, flush=True)
    # ... 其他代码 ...
    print(f"DEBUG: 调用 get_processing_status, 参数: project_id={project_id}", file=sys.stderr)
    status = processing_service.get_processing_status(project_id)
    print(f"DEBUG: get_processing_status 返回成功", file=sys.stderr)
except Exception as e:
    import traceback
    error_msg = f"ERROR in get_processing_status: {e}\n{traceback.format_exc()}"
    print(error_msg, file=sys.stderr)
    logger.error(error_msg)
```

**修复后代码**:
```python
logger.debug(f"get_processing_status: project_id = {project_id}")
try:
    logger.debug(f"processing_service = {processing_service}, type = {type(processing_service)}")
    # ... 其他代码 ...
    logger.debug(f"调用 get_processing_status, 参数: project_id={project_id}")
    status = processing_service.get_processing_status(project_id)
    logger.debug("get_processing_status 返回成功")
except Exception as e:
    import traceback
    error_msg = f"ERROR in get_processing_status: {e}\n{traceback.format_exc()}"
    logger.error(error_msg)
```

**修复说明**:
- 移除了不必要的 `import sys` 语句
- 将所有 `print` 语句替换为 `logger.debug` 语句
- 保留了 `logger.error` 语句用于错误记录
- 使用结构化日志系统而不是直接打印到 stderr，便于日志管理和级别控制

### 问题3：监控项目导出功能调用不存在的方法，导致 500 错误

**文件位置**: `monitor/export_manager.py:385-471`

**问题描述**: 
在 `get_export_data` 方法中，代码尝试调用 `data_manager.get_all_data()` 方法，但 `DataManager` 类中不存在这个方法。这会导致 `AttributeError`，进而返回 500 内部服务器错误。

**错误现象**:
- 前端请求 `/monitor-api/export/data` 时返回 500 错误
- 控制台显示: `❌ Monitor API Error: AxiosError`
- 导出功能完全不可用

**原始代码**:
```python
def get_export_data(self, ...):
    # ...
    for room_id in rooms:
        # 从数据管理器获取数据
        if hasattr(self.data_manager, 'get_data_by_room_and_date'):
            if start_date and end_date:
                # 如果有明确的日期范围
                room_data = self.data_manager.get_data_by_room_and_date(room_id, start_date)
                # 这里简化处理，实际应该处理日期范围
            else:
                # 获取所有数据
                room_data = self.data_manager.get_all_data()  # ❌ 这个方法不存在！
                # 按房间过滤
                room_data = [item for item in room_data if item.get('room_id') == room_id]
        else:
            # 备用方式：获取所有数据
            room_data = self.data_manager.get_all_data()  # ❌ 这个方法不存在！
            # 按房间过滤
            room_data = [item for item in room_data if item.get('room_id') == room_id]
```

**修复后代码**:
```python
def get_export_data(self, ...):
    # ...
    for room_id in rooms:
        room_data = []
        
        # 从数据管理器获取数据
        if start_date and end_date:
            # 处理日期范围 - 正确遍历日期范围内的每一天
            try:
                start_dt = datetime.strptime(start_date, '%Y-%m-%d')
                end_dt = datetime.strptime(end_date, '%Y-%m-%d')
                
                # 遍历日期范围获取数据
                current_dt = start_dt
                while current_dt <= end_dt:
                    date_str = current_dt.strftime('%Y-%m-%d')
                    if hasattr(self.data_manager, 'get_data_by_room_and_date'):
                        day_data = self.data_manager.get_data_by_room_and_date(room_id, date_str)
                        room_data.extend(day_data)
                    elif hasattr(self.data_manager, 'get_data_by_date'):
                        day_data = self.data_manager.get_data_by_date(date_str)
                        room_data.extend([item for item in day_data if item.get('room_id') == room_id])
                    current_dt += timedelta(days=1)
            except Exception as e:
                self.logger.warning(f"处理日期范围失败: {e}")
                # 回退到只获取开始日期的数据
                if hasattr(self.data_manager, 'get_data_by_room_and_date'):
                    room_data = self.data_manager.get_data_by_room_and_date(room_id, start_date)
                elif hasattr(self.data_manager, 'get_data_by_date'):
                    day_data = self.data_manager.get_data_by_date(start_date)
                    room_data = [item for item in day_data if item.get('room_id') == room_id]
        else:
            # 没有指定日期范围，获取最近30天的数据
            if hasattr(self.data_manager, 'get_data_by_room'):
                room_data = self.data_manager.get_data_by_room(room_id, days=30)
            elif hasattr(self.data_manager, 'get_recent_days_data'):
                all_days_data = self.data_manager.get_recent_days_data(days=30)
                # 合并所有日期的数据
                for date_data in all_days_data.values():
                    room_data.extend([item for item in date_data if item.get('room_id') == room_id])
```

**修复说明**:
1. **移除了不存在的方法调用**: 删除了对 `get_all_data()` 的调用
2. **正确处理日期范围**: 当有明确的开始和结束日期时，正确遍历日期范围内的每一天
3. **添加回退机制**: 如果日期范围处理失败，回退到只获取开始日期的数据
4. **正确处理无日期范围的情况**: 使用 `get_data_by_room()` 或 `get_recent_days_data()` 获取最近30天的数据
5. **添加异常处理**: 使用 `try-except` 捕获可能的异常

**根本原因分析**:
- 代码尝试调用 `DataManager` 类中不存在的 `get_all_data()` 方法
- 日期范围处理不正确，只获取了开始日期的数据，而没有遍历整个范围
- 缺少异常处理和回退机制

### 问题4：导出数据中时间格式和房间ID格式不正确

**文件位置**: `monitor/export_manager.py:464-496`

**问题描述**: 
导出的 Excel/CSV 文件中，时间显示为原始的时间戳（如 `1,777,720,585.00`），而不是格式化的日期时间字符串。房间ID显示为带千位分隔符的数字（如 `22,391,541`）。

**错误现象**:
- 导出的 Excel 文件中，时间列显示为 `1,777,720,585.00`
- 房间ID列显示为 `22,391,541`（带千位分隔符）
- 用户无法直接识别时间信息

**原始代码**:
```python
# 格式化数据
for item in room_data:
    formatted = {}
    
    # 基础指标
    if metrics.get('basic'):
        formatted.update({
            '时间': item.get('time_display', item.get('timestamp', '-')),
            '用户': item.get('username', '未知用户'),
            '内容': item.get('content', '-')
        })
    
    # 评分指标
    if metrics.get('rating'):
        formatted['评分'] = item.get('rating', 0)
    
    # 房间信息
    if metrics.get('room'):
        formatted['房间ID'] = room_id
```

**修复后代码**:
```python
# 格式化数据
for item in room_data:
    formatted = {}
    
    # 格式化时间戳
    time_display = item.get('time_display')
    if time_display is None:
        timestamp = item.get('timestamp')
        if timestamp is not None:
            try:
                timestamp_float = float(timestamp)
                dt = datetime.fromtimestamp(timestamp_float)
                time_display = dt.strftime('%Y-%m-%d %H:%M:%S')
            except (ValueError, TypeError):
                time_display = '-'
        else:
            time_display = '-'
    
    # 基础指标
    if metrics.get('basic'):
        formatted.update({
            '时间': time_display,
            '用户': item.get('username', '未知用户'),
            '内容': item.get('content', '-')
        })
    
    # 评分指标
    if metrics.get('rating'):
        formatted['评分'] = item.get('rating', 0)
    
    # 房间信息（确保为整数，不带千位分隔符）
    if metrics.get('room'):
        formatted['房间ID'] = int(room_id)
```

**修复说明**:
1. **添加时间格式化逻辑**: 
   - 检查 `time_display` 是否存在
   - 如果不存在，获取 `timestamp` 并转换为浮点数
   - 使用 `datetime.fromtimestamp()` 转换为日期时间对象
   - 格式化为 `YYYY-MM-DD HH:MM:SS` 字符串
   - 添加异常处理，防止无效的时间戳

2. **确保房间ID为整数**:
   - 使用 `int(room_id)` 确保房间ID为整数类型
   - 避免在 Excel 中被格式化为带千位分隔符的数字

**根本原因分析**:
- 代码直接使用 `item.get('time_display', item.get('timestamp', '-'))`，当 `time_display` 不存在时，直接使用原始的时间戳浮点数
- 时间戳浮点数在 Excel 中被自动格式化为带千位分隔符的数字
- 房间ID虽然是整数，但没有显式转换，可能在某些情况下被格式化为带千位分隔符的数字
